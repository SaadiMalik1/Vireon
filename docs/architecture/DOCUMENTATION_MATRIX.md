# VIREON Ecosystem Documentation Matrix

This matrix establishes the definitive Single Source of Truth (SSoT) for all documentation across the VIREON ecosystem. 

All duplicate README sections, redundant installation instructions, and fragmented design documents across repositories MUST be replaced with explicit links to these central paths.

## Documentation Hierarchy & Ownership

| Category | SSoT Path | Owner Repository | Description |
| :--- | :--- | :--- | :--- |
| **Architecture** | `docs/architecture/` | `workspace` | Master ecosystem design, boundaries, layering. |
| **Reference** | `docs/reference/` | `workspace` | Ecosystem-wide glossaries, threat models, and standards. |
| **API Docs (Python)** | `docs/api/python/` | `vireon` | Autogenerated from Python docstrings via Sphinx/MkDocs. |
| **API Docs (Rust)** | `docs/api/rust/` | `neurodsl` | Autogenerated via `cargo doc`. |
| **SDK Guide** | `docs/sdk/` | `vireon` | Guides on writing custom plugins and providers against the SDK. |
| **Tutorials** | `docs/tutorials/` | `workspace` | End-to-end multi-repo tutorials (e.g., "Building your first Digital Twin"). |
| **User Guides** | `docs/guides/` | `vireon-lab` | Instructions for operating the UI and CLI applications. |
| **ADRs / RFCs** | `docs/architecture/decisions/` | `workspace` | Centralized repository for all Architecture Decision Records and RFCs. |
| **Benchmarks** | `docs/validation/benchmarks/` | `neurodsl` | Rust engine benchmark reports and deterministic reproducibility instructions. |
| **Validation** | `docs/validation/` | `workspace` | Clinical/FDA validation records, AT constraints. |
| **Integration** | `docs/integration/` | `vireon` | Third-party vendor integration and telemetry ingestion guides. |
| **Deployment / Operations**| `docs/ops/` | `vireon-lab` | Docker Compose instructions, scaling, and operational monitoring. |
| **Governance** | `docs/governance/` | `.github` | Contribution guidelines, Code of Conduct, security reporting. |
| **Release / Roadmap** | `docs/roadmap/` | `workspace` | Ecosystem release schedules, changelogs, and 5-year visions. |
| **Knowledge Base** | `docs/knowledge/` | `workspace` | Centralized medical device standards (ISO 14971, IEC 62304) and neuroscience protocols. |

## Deprecation & Deduplication Mandate

The following legacy documentation structures must be strictly replaced with redirects to the paths above:
1. Delete `vireon/docs/getting_started.md` -> Redirect to `workspace/docs/tutorials/`.
2. Delete `vireon-lab/README.md` "Architecture" section -> Redirect to `workspace/docs/architecture/`.
3. Delete `neurodsl/ARCHITECTURE.md` -> Move contents into `workspace/docs/architecture/rust_engine/`.
4. Delete decentralized `CONTRIBUTING.md` files in every repo -> Soft-link to `.github/CONTRIBUTING.md`.
5. Remove fragmented Docker instructions in `vireon` and `vireon-lab` -> Consolidate into `workspace/docs/ops/docker.md`.

## Quality Enforcement
- CI pipelines will strictly fail if multiple `CONTRIBUTING.md` or architectural documents are detected in sub-repositories.
- Docstrings are the only permitted source of truth for API contracts. No manual Markdown API documentation is permitted.
