# VIREON Governance Model

This document outlines the governance structure for the VIREON open-source ecosystem.

## 1. Technical Steering Committee (TSC)
The TSC is the highest decision-making body, responsible for overarching architectural consistency, merging constraints (`AT-01` to `AT-08`), and the approval of new `vireon.sdk` API versions. 

## 2. Decision Making
All major architectural changes must be submitted as an Architecture Decision Record (ADR) and achieve a 2/3 majority vote within the TSC.

## 3. Subprojects
Each submodule (`neurodsl`, `vireon-lab`, `vireon`) has designated Maintainers who have autonomy over implementation details as long as they adhere strictly to the ecosystem contracts and the `DOCUMENTATION_MATRIX.md`.
